v2.1.81 (2026-03-20)

最新リリース。主な変更点:

  • --bare フラグ追加: スクリプト用 -p 呼び出し向け。hooks、LSP、plugin sync、skill directory walks をスキップ。ANTHROPIC_API_KEY または --settings 経由の apiKeyHelper が必要（OAuth/keychain認証は無効）。auto-memory も完全無効化
  • --channels パーミッションリレー: permission capability を宣言したチャンネルサーバーがツール承認プロンプトをスマホに転送可能に
  • バグ修正: 複数セッション同時実行時のOAuthトークン再認証問題、voice modeのリトライ失敗時のエラー表示、WebSocket接続切断からの音声復旧、Node.js 18でのクラッシュ、Bashコマンド内ダッシュによる不要なパーミッションプロンプトなど多数
  • 改善: MCP read/search ツール呼び出しを「Queried {server}」に集約（Ctrl+Oで展開）、! bashモードの発見性向上、ref-tracked pluginの再clone動作、Remote Controlセッションタイトルの更新
  • MCP OAuth更新: Client ID Metadata Document (CIMD / SEP-991) サポート
  • 変更: plan modeで「clear context」オプションをデフォルト非表示に（"showClearContextOnPlanAccept": trueで復元可）
  • Windows: WSL含むWindows Terminalで行単位レスポンスストリーミングを無効化（レンダリング問題のため）
  • VSCode: Git Bash使用時のWindows PATH継承を修正（v2.1.78でのリグレッション）

v2.1.80 (2026-03-19)

  • rate_limits フィールド追加: statuslineスクリプト向け。Claude.aiのレート制限使用状況（5時間/7日ウィンドウ、used_percentage、resets_at）を表示可能
  • --channels (research preview): MCPサーバーがセッションにメッセージをプッシュ可能に
  • plugin marketplace source: settings.json内でpluginエントリをインライン宣言可能に（source: 'settings'）
  • effort frontmatter: skills/slash commandsでモデルeffortレベルをオーバーライド可能
  • バグ修正: --resume時の並列ツール結果消失、voice mode WebSocket失敗（Cloudflare bot検出）、fine-grained tool streaming時の400エラーなど
  • パフォーマンス: 大規模リポジトリでの起動時メモリ使用量を約80MB削減（250kファイルリポ）

v2.1.78 (2026-03-17)

  • StopFailure hook: APIエラー（レート制限、認証失敗等）でターンが終了した時に発火
  • ${CLAUDE_PLUGIN_DATA} 変数: plugin永続状態用。plugin更新後も維持、uninstall時に削除確認
  • 行単位レスポンスストリーミング: 生成中のテキストが行単位でストリーム表示
  • セキュリティ修正: sandbox.enabled: true 設定時に依存関係が不足している場合、サイレント無効化ではなく起動時警告を表示
  • tmux対応: set -g allow-passthrough on でターミナル通知が外部ターミナルに到達
  • ANTHROPIC_CUSTOM_MODEL_OPTION: /model ピッカーにカスタムエントリを追加可能

v2.1.77 (2026-03-17)

  • Claude Opus 4.6 出力トークン上限引き上げ: デフォルト64kトークン、上限128kトークン（Opus 4.6/Sonnet 4.6）
  • allowRead sandbox設定: denyRead 領域内で読み取りアクセスを再許可可能
  • /copy N: N番目の最新アシスタント応答をコピー可能
  • バグ修正: compound bashコマンドの「Always Allow」ルール保存、auto-updaterのメモリ蓄積、--resume時の会話履歴切り詰め、PreToolUse hooks が deny ルールをバイパスする問題など多数
  • パフォーマンス: macOSで起動約60ms高速化、--resume が最大45%高速化・ピークメモリ約100-150MB削減
  • Agent tool変更: resume パラメータ廃止、SendMessage({to: agentId}) を使用
  • /fork を /branch にリネーム（/fork はエイリアスとして継続）

v2.1.76 (2026-03-14)

  • MCP elicitation サポート: MCPサーバーがタスク中にインタラクティブダイアログ（フォームフィールドまたはブラウザURL）で構造化入力を要求可能
  • Elicitation / ElicitationResult hooks: レスポンスを送信前にインターセプト/オーバーライド可能
  • -n / --name フラグ: 起動時にセッション表示名を設定
  • worktree.sparsePaths 設定: 大規模モノレポで git sparse-checkout を使用して必要なディレクトリのみチェックアウト
  • PostCompact hook: compaction完了後に発火
  • /effort コマンド: モデルeffortレベルを設定
